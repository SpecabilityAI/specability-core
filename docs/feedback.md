# Feedback

Specability Core does not send telemetry automatically. Useful feedback is
explicit, selected, and reviewable before it leaves your environment.

## Good Feedback

The most useful feedback describes a real workflow:

- Agent host: Codex, Claude Code, Gemini CLI, or another host.
- Task type.
- Expected outcome.
- Where the agent lost reliability.
- What evidence existed: tests, review traces, runtime records, logs, or manual
  checks.
- What Specability should encode as a playbook, spec, or workflow rule.

Do not submit confidential source code, secrets, customer data, private runtime
databases, or logs that contain sensitive content.

## Issue Types

Use:

- Bug report: install or runtime failure.
- Workflow feedback: real workflow experience.
- Workflow request: request for new playbook/spec support.
